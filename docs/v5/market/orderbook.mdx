Query for orderbook depth data.
> **Covers: Spot**

* Spot: 1000-level of orderbook data

:::info
* The response is in the snapshot format.
* [Retail Price Improvement (RPI)](https://www.abfinance.com/en/help-center/article/Retail-Price-Improvement-RPI-Order) orders will not be included in the response message and will not be visible over API.
:::

### HTTP Request
GET `/v5/market/orderbook`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type. `spot` |
|symbol |**true** |string |Symbol name, like `BTCUSDT`, uppercase only|
|limit |false |integer |Limit size for each bid and ask<ul><li>`spot`: [`1`, `200`]. Default: `1`.</li></ul>|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|s |string |Symbol name |
|b |array |Bid, buyer. Sorted by price in descending order |
|> b[0] |string |Bid price |
|> b[1] |string |Bid size |
|a |array |Ask, seller. Sorted by price in ascending order |
|> a[0] |string |Ask price |
|> a[1] |string |Ask size |
|ts |integer |The timestamp (ms) that the system generates the data |
|u |integer |Update ID, is always in sequence<ul><li>For spot, corresponds to `u` in the 1000-level [WebSocket orderbook stream](https://bybit-exchange.github.io/docs/v5/websocket/public/orderbook)</li></ul> |
|seq |integer |Cross sequence <li>You can use this field to compare different levels orderbook data, and for the smaller seq, then it means the data is generated earlier. </li>|
|cts |integer |The timestamp from the matching engine when this orderbook data is produced. It can be correlated with `T` from [public trade channel](../websocket/public/trade#)|
