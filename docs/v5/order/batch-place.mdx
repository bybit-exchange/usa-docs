:::tip
This endpoint allows you to place more than one order in a single request.
- Make sure you have sufficient funds in your account when placing an order. Once an order is placed, according to the
funds required by the order, the funds in your account will be frozen by the corresponding amount during the life cycle
of the order.
- A maximum of 10 orders (spot) can be placed per request. The returned data list is divided into two lists.
The first list indicates whether or not the order creation was successful and the second list details the created order information. The structure of the two lists are completely consistent.
:::

:::info
- **Spot rate limit instruction**, please check [here](../rate-limit#instructions-for-batch-endpoints)
- **Risk control limit notice:**<br/>
We will monitor on your API requests. When the total number of orders of a single user (aggregated the number of orders across main account and subaccounts) within a day (UTC 0 - UTC 24) exceeds a certain upper limit, the platform will reserve the right to remind, warn, and impose necessary restrictions.
Customers who use API default to acceptance of these terms and have the obligation to cooperate with adjustments.
:::

### HTTP Request
POST `/v5/order/create-batch`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type`spot`|
|request |**true** |array |Object |
|> symbol |**true** |string |Symbol name, like `BTCUSDT`, uppercase only |
|> side |**true** |string |`Buy`, `Sell`|
|> [orderType](../enum#ordertype) |**true** |string |`Market`, `Limit`|
|> qty |**true** |string |Order quantity <ul><li>Spot: set `marketUnit` for market order qty unit, `quoteCoin` for market buy by default, `baseCoin` for market sell by default</li></ul>|
|> marketUnit |false |string |The unit for `qty` when create **Spot market** orders, `orderFilter`="tpslOrder" and "StopOrder" are supported as well.<li>`baseCoin`: for example, buy BTCUSDT, then "qty" unit is BTC</li><li>`quoteCoin`: for example, sell BTCUSDT, then "qty" unit is USDT</li>|
|> price |false |string |Order price <ul><li>Market order will ignore this field</li><li>Please check the min price and price precision from [instrument info](../market/instrument#response-parameters) endpoint</li></ul> |
|> orderFilter |false |string |If it is not passed, `Order` by default. <ul><li>`Order`</li><li>`tpslOrder`: Spot TP/SL order, the assets are occupied even before the order is triggered</li><li>`StopOrder`: Spot conditional order, the assets will not be occupied until the price of the underlying asset reaches the trigger price, and the required assets will be occupied after the Conditional order is triggered</li></ul>Valid for `spot` **only**|
|> triggerPrice |false |string |<ul><li>For spot, it is the `orderFilter`="tpslOrder", or "StopOrder" trigger price</li></ul>|
|> [triggerBy](../enum#triggerby) |false |string |Conditional order param (Perps & Futures). Trigger price type. `LastPrice`, `IndexPrice`, `MarkPrice`|
|> [timeInForce](../enum#timeinforce) |false |string |<a href="https://www.abfinance.com/en/help-center/article/What-Are-Time-In-Force-TIF-GTC-IOC-FOK">Time in force</a> <ul><li>Market order will use `IOC` directly</li><li>If not passed, `GTC` is used by default</li> </ul>|
|> orderLinkId |false |string |User customised order ID. A max of 36 characters. Combinations of numbers, letters (upper and lower cases), dashes, and underscores are supported.<br/>Spot: orderLinkId rules:<br/><ul><li>optional param</li><li>always unique</li></ul>|
|> takeProfit |false |string |Take profit price|
|> stopLoss |false |string |Stop loss price|
|> [smpType](../enum#smptype) |false |string |Smp execution type. [What is SMP?](../smp)|
|> tpLimitPrice |false |string |The limit order price when take profit price is triggered <ul><li>Spot: it is required when the order has `takeProfit` and `tpOrderType=Limit`</li></ul>|
|> slLimitPrice |false |string |The limit order price when stop loss price is triggered<ul><li>Spot: it is required when the order has `stopLoss` and `slOrderType=Limit`</li></ul>|
|> tpOrderType |false |string |The order type when take profit is triggered <ul><li>Spot: <br/>`Market`: when you set "takeProfit", <br/>`Limit`: when you set "takeProfit" and "tpLimitPrice" </li></ul>|
|> slOrderType |false |string |The order type when stop loss is triggered <ul><li>Spot: <br/>`Market`: when you set "stopLoss", <br/>`Limit`: when you set "stopLoss" and "slLimitPrice" </li></ul>|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|result |Object | |
|> list |array |Object |
|>> category |string |Product type |
|>> symbol |string |Symbol name |
|>> orderId |string |Order ID |
|>> orderLinkId |string |User customised order ID |
|>> createAt |string |Order created time (ms) |
|retExtInfo |Object | |
|> list |array |Object |
|>> code |number |Success/error code |
|>> msg |string |Success/error message |

:::info
The acknowledgement of an place order request indicates that the request was sucessfully accepted. This request is asynchronous so please use the websocket to confirm the order status.
:::
