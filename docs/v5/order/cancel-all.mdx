Cancel all open orders

:::info
- Support cancel orders by `symbol`/`baseCoin`. If you pass multiple of these params, the system will process one of param, which priority is `symbol` > `baseCoin`.
- **NOTE**: category=_spot_, you can cancel all spot open orders (normal order by default) without passing other params.
:::

:::info
**Spot**: no limit<br/>
:::

### HTTP Request
POST `/v5/order/cancel-all`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type.`spot`|
|symbol |false|string |Symbol name, like `BTCUSDT`, uppercase only|
|baseCoin |false|string |Base coin, uppercase only|
|orderFilter |false |string |<ul><li>category=`spot`, you can pass `Order`, `tpslOrder`, `StopOrder`, `OcoOrder`, `BidirectionalTpslOrder`<br/>If not passed, `Order` by default</li></ul>|

:::info
The acknowledgement of create/amend/cancel order requests indicates that the request was sucessfully accepted. The request is asynchronous so please use the websocket to confirm the order status.
:::


---

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|list |array |Object|
|> orderId |string |Order ID |
|> orderLinkId |string |User customised order ID |
|success |string |"1": success, "0": fail|
